gen_media · chosen UX — Canvas-First

The canvas is the app

Final direction. One continuous, zoomable space holding the whole exploration tree. Selecting a node reveals its actions; precision gestures happen by zooming into the node itself — the map's zoom is the magnifier. Every operation, generative or deterministic, creates child nodes. Three states below show the complete loop.

decided: Canvas-First decided: zoom-to-edit regions decided: flat tool menu, ✦ marks AI ops decided: pick-a-node references

Gesture model

The full input vocabulary — small on purpose.

GestureResult
click nodeSelect — action menu appears on the node, inspector opens with details + op controls
drag nodeReposition it (positions persist)
drag background / scrollPan / zoom the canvas (React Flow defaults, minimap bottom-left)
double-click nodeZoom-to-fit that node — camera animates until the node fills most of the view
drag on selected image (region op armed)Draw the inpaint / crop rectangle directly on the image; drag text to place it
EscDisarm region op, else deselect
type in bottom prompt barGenerate a new root image (new tree on the open canvas); Upload button sits beside it

One grammar, two tool families

Every tool — AI or instant — runs the same three-beat loop. What differs is only the middle beat and the wait. In the menu, ✦ marks a model call (~seconds, returns N variants); unmarked verbs are instant, free, and previewed live.

1 · Arm

Pick a verb on the selected node: ✦ Edit · ✦ Inpaint · ✦ Variations · Crop · Resize · Text

2 · Configure on the image

AI: write the prompt, drag a region, attach a reference. Instant: drag crop handles or place text — live preview.

3 · Commit → child nodes

✦ Run creates N pending children that fill in as results land. Instant Apply creates one finished child immediately.

Why this state matters

  • The answer to whack-a-mole is structural: imprecise models become tolerable when every attempt is a cheap, comparable, discardable branch — and inpaint guarantees pixels outside the region never change.
  • Nothing blocks: while v8 generates you can select v7, zoom in, and keep working. Ops are fire-and-forget.
  • Errors are nodes too — the failed op keeps its recipe, so Retry is one click, and the tree honestly records what you tried.

Considered and set aside: Studio Split (big stage + tree sidebar) — precision was great but it demoted the branching tree to a layers panel; Map & Focus (canvas + full-screen focus mode) — solved both, but zoom-to-edit achieves the same inside one continuous space with no mode switch. Canvas-First with zoom-to-edit is the chosen direction.
